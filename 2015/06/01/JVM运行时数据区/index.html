JVM运行时数据区

发表于 2015-06-01 | 分类于 Java ， JVM

JVM运行时数据区如上图所示：其中方法区和堆由所有线程共享；虚拟机栈，本地方法栈和程序计数器是线程似有的。

# 堆

堆（heap）和方法区(Method Area)被JVM内的所有线程共享，堆中存放的是类的对象和数组。

# 方法区

方法区(Method Area)用于储存被虚拟机加载的类信息，常量，即时编译后的代码等，类似于其它语言中存放编译后代码的区域以及操作系统中线程的text字段。

# # 运行时常量池

运行时常量池用于存放编译期生成的各种字面量和符号引用，这些字面量和符号引用在程序运行时用到。
堆和方法区都是在JVM启动时就创建

# 虚拟机栈

虚拟机栈（Virtual Machine Stacks）是每个线程私有的，在线程被创建的同时创建一个该线程私有的虚拟机栈，JVM中的虚拟机栈类似于C语言中的栈，虚拟机栈中存放局部变量、操作数和方法出口等。

# 程序计数器（PC）

每个线程都拥有一个程序计数器，各线程的程序计数器相互独立，如果线程执行的是Java方法，程序计数器指向正在执行的字节码指令地址，如果当前执行的是Native方法，程序计数器的值是未定义（Undifined）的。

# 本地方法栈

如果虚拟机支持本地方法，线程启动时会创建一个本地方法栈，本地方法栈支持Java语言之外的语言运行。
